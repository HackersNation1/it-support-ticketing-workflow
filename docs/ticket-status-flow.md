# Ticket Status Flow

This document explains the lifecycle of an IT support ticket
from creation to closure.

---

## Status Definitions

### New
- Ticket has been submitted via Google Form
- No action taken yet

### In Progress
- Issue has been reviewed
- Diagnosis or troubleshooting has started

### Waiting for Payment
- Issue has been identified
- User has been informed of cost
- No further action until payment is confirmed

### Resolved
- Issue has been fixed
- System tested and confirmed working

### Closed
- Ticket confirmed completed
- No further action required
